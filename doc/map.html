inurl:holidays.ttl filetype:ttl

in google returned nothing, changing ttl to rdf revealed 5 documents, none of them a list of holidays. finally trying

"april fools day" filetype:rdf christmas

is closer, one document with holiday names but no dates, and another with a selection of dates in 2009

holidays csv easily turned up the goods

as nice of an idea as RDF lovingly crafted may be, crude data scraped or converted remains the norm. agreement on schemas and property-names seems even more remote, from Cyc, Schema.org, Freebase, RDF Vocabulary Markets, Microformats to bare words as JSON or XML keys linking no schema at all

map those property-names to something useful

view=map

crucial selections from SIOC and Dublin Core are available.

submitting creates a query string like

filter=map&Date=http://purl.org/dc/terms/date&Holiday=http://purl.org/dc/terms/date

possible to construct by hand, if you are a 'power user'. adding view=year now has clean input for an existing view which expects DC terms
